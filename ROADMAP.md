# 项目路线图 (Product Roadmap)

本文档用于记录 **Partner App (咔哒 ⚡ 闪电搭子)** 的后续开发计划与优化方向。

## 🟢 当前状态 (Current Status - v1.1)
- [x] **核心闭环**: 发布任务 -> 实时广播 -> 抢单(阅后即焚)。
- [x] **无登录机制**: 基于本地 Token 的身份识别。
- [x] **续命系统**: 发布者可手动重置任务时间（15分钟）。
- [x] **顶替机制**: 限制单人单任务，发新自动删旧。
- [x] **数据持久化**: 接入 MongoDB，防止服务器重启数据丢失。
- [x] **UI/UX**: 悬浮发布按钮，深色模式。
- [x] **响应式布局**: PC 端三列网格布局，移动端 App 级体验。
- [x] **🏷️ 标签+属性系统**: 
    - 支持按游戏分区。
    - 支持大区、模式双维度分行筛选。
    - 发布弹窗动态属性选择。

---

## 🚀 短期目标 (v1.2 - 核心体验深化)
> *优先级：P0 (最高)*

- [ ] **🔔 新消息提示音 & 通知**
    - 目标：允许用户挂后台等待。
    - 方案：申请浏览器 Notification 权限；有新任务时播放简短音效（"叮"）。
- [ ] **📱 移动端深度适配 (PWA)**
    - 目标：提供接近原生 App 的体验。
    - 方案：增加 PWA 支持（可添加到主屏幕）。
- [ ] **💬 预设发布文案**
    - 目标：降低发布门槛，减少打字。
    - 方案：提供热门模板（"大乱斗缺1"、"云顶双人"等）一键填入。

---

## 🛡️ 中期目标 (v1.2 - 安全与治理)
> *优先级：P1 (重要)*

- [ ] **🚫 简单的反垃圾/限流 (Rate Limiting)**
    - 目标：防止恶意刷屏或脚本攻击。
    - 方案：限制同 IP 发布频率（如：1分钟/次）。
- [ ] **😶 敏感词过滤**
    - 目标：合规化，防止违规/诈骗信息。
    - 方案：后端接入敏感词库，命中则拒绝发布或仅自己可见。
- [ ] **🚨 举报机制**
    - 目标：社区自我净化。
    - 方案：任务卡片增加"举报"按钮，后台记录并自动标记高风险用户。

---

## 🌟 长期愿景 (v2.0 - 社交与运营)
> *优先级：P2 (锦上添花)*

- [ ] **👋 轻互动 ("戳一戳")**
    - 目标：解决社恐不敢直接加好友的问题。
    - 方案：允许对任务"戳一下"，发布者收到提醒，增加连接机会。
- [ ] **🟢 在线状态感知**
    - 目标：避免联系已下线的死号。
    - 方案：基于 Socket 连接状态，在名字旁显示"在线/离线"。
- [ ] **📊 数据统计后台 (/admin)**
    - 目标：运营决策辅助。
    - 方案：查看每日 DAU、发布量、平均抢单耗时等数据。
- [ ] **📢 全局公告栏**
    - 目标：运营通知。
    - 方案：顶部跑马灯，支持后端动态配置。

---

## 💡 灵感池 (Backlog / Ideas)
> *待定想法，需验证需求*

- **区域/服务器筛选**: 针对不仅分游戏还分大区的情况（如 LOL 电一）。
- **黑名单系统**: 用户可以在本地屏蔽某些讨厌的发布者。
- **声音/语音条发布**: 允许录制 5秒 语音需求（技术成本较高，慎重）。
- **简单的积分/成就系统**: "搭子王"称号（暂不考虑，需账号系统）。
